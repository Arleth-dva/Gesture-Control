# Documentación del Proyecto — Carpeta /docs

Esta carpeta contiene toda la documentación oficial del proyecto **GestureControl**, organizada en diferentes bitácoras y archivos complementarios. Cada documento registra el progreso, los problemas detectados y las soluciones implementadas durante el desarrollo del sistema.

## Contenido de la carpeta

### Bitácoras del Desarrollo

Documentos que registran de forma cronológica el trabajo realizado:

* **Bitácora 1**
  Configuración inicial, análisis del proyecto, primeros errores de dependencias y reestructuración de la base del sistema.

* **Bitácora 2**
  Etapa de depuración, reorganización de módulos, diagnóstico de errores de carga y establecimiento del primer detector funcional.

* **Bitácora 3**
  Integración completa del sistema: soluciones definitivas de video, cambio de runtime, reescritura de módulos y detección estable.

### Bitácora de Publicación en Redes Sociales

Registro del proceso de divulgación del proyecto, diseño de publicaciones.

### Bitácora de Creación del Mundo Virtual

Documentación de la construcción del entorno virtual que se integrará con GestureControl, incluyendo diseño, pruebas y ajustes.

### Artículo del Proyecto 

Documento de carácter técnico y formal que explica en detalle la solución, arquitectura, metodología y resultados del proyecto.

---

## Propósito de la carpeta

La carpeta **/docs** funciona como un repositorio histórico y técnico del proyecto.
Su objetivo es:

* Mantener un **registro claro y completo** del desarrollo.
* Servir como **guía de referencia** para futuros colaboradores.
* Documentar las **decisiones técnicas** importantes.
* Facilitar la presentación del proyecto en contextos académicos o profesionales.
